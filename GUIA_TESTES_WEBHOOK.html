🔗 Guia de Testes — Webhook PyFlow RPA

Testando os dois sentidos: enviar dados para fora e receber dados de fora.

📤 PARTE 1 — Enviar para Webhook externo

PyFlow faz POST para uma URL de fora (Zapier, Make, n8n, Discord, ou qualquer sistema).

Passo 1 — Criar uma URL de teste gratuita

1
Abra o site webhook.site

Ao entrar, você recebe uma URL única automaticamente. Ex: https://webhook.site/abc12345-...

2
Copie a URL gerada

Ela fica no topo da página. Parece: https://webhook.site/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

3
No PyFlow — abra o fluxo teste_send_webhook

Clique no bloco Enviar para Webhook → campo URL do Webhook → cole a URL do webhook.site

4
Execute o fluxo no PyFlow

Clique ▶ Executar. O bloco vai fazer POST com os dados.

5
Veja o resultado no webhook.site

A página atualiza automaticamente e mostra o payload que chegou:

{
  "produto": "Notebook Dell XPS",
  "preco":   "4999.90",
  "origem":  "PyFlow RPA",
  "timestamp": "agora"
}
✅ Se aparecer no webhook.site — funcionou! O PyFlow enviou dados para fora com sucesso.

Payload personalizável — como funciona

No campo Payload JSON do bloco você pode usar {{variavel}} para incluir qualquer variável do contexto:

{
  "nome":    "{{nome_cliente}}",   ← variável do contexto
  "preco":   "{{preco_total}}",   ← variável do contexto
  "fixo":    "PyFlow RPA",        ← valor fixo
  "lista":   "{{itens_extraidos}}" ← lista de scraping
}

A resposta do servidor fica na variável configurada em Salvar resposta (padrão: webhook_resposta).

📥 PARTE 2 — Receber Webhook (o PyFlow como servidor)

Um sistema externo (ou você mesmo via curl/Postman) chama o PyFlow e o fluxo dispara automaticamente.

Como funciona a rota

MétodoURLO que faz
POST /webhook/{nome_do_fluxo} Recebe payload JSON e dispara o fluxo
GET /webhook/inbox Mostra os últimos payloads recebidos (debug)

O payload é injetado automaticamente como variáveis: cada campo vira webhook_CAMPO e o dict completo fica em webhook_payload.

Passo a passo — Testar com PowerShell (sem instalar nada)

1
Abra o PyFlow e confirme que o servidor está rodando

Olhe na barra inferior — deve mostrar API: http://127.0.0.1:8080 ou similar.

2
Abra o PowerShell (Win + R → powershell → Enter)
3
Cole e execute este comando: 
# Envia POST para o PyFlow disparar o fluxo "teste_webhook_receber"
$body = @{
    nome  = "Gleidson"
    valor = "42"
    acao  = "teste_webhook"
} | ConvertTo-Json

Invoke-RestMethod `
    -Uri "http://127.0.0.1:8080/webhook/teste_webhook_receber" `
    -Method POST `
    -Body $body `
    -ContentType "application/json"
4
Observe no PyFlow

O fluxo teste_webhook_receber vai disparar automaticamente. A janela de mensagem vai aparecer com os dados que você enviou.

5
Resposta esperada do PowerShell: 
{
  "success":    true,
  "message":    "Webhook recebido — fluxo 'Teste — Receber Webhook' disparado.",
  "flow":       "Teste — Receber Webhook",
  "payload":    { "nome": "Gleidson", "valor": "42" },
  "variables":  ["webhook_nome", "webhook_valor", "webhook_acao", "webhook_payload"]
}

Alternativa — Testar com curl (se tiver instalado)

curl -X POST http://127.0.0.1:8080/webhook/teste_webhook_receber ^
  -H "Content-Type: application/json" ^
  -d "{\"nome\": \"Gleidson\", \"valor\": \"42\"}"

Alternativa — Testar pelo Swagger UI (sem precisar de terminal)

1
Abra no navegador: http://127.0.0.1:8080/docs

A página do Swagger já lista todos os endpoints, incluindo POST /webhook/{flow_name}

2
Clique em POST /webhook/{flow_name} → "Try it out"

Preencha flow_name = teste_webhook_receber e o body JSON

3
Clique "Execute" — o fluxo dispara no PyFlow

🔄 PARTE 3 — Variáveis disponíveis no fluxo após receber webhook

Se você enviou este payload:

{ "nome": "Gleidson", "valor": "42", "acao": "cadastrar" }

Estas variáveis ficam disponíveis no fluxo via {{variavel}}:

VariávelValorDescrição
{{webhook_nome}}"Gleidson"Campo "nome" do payload
{{webhook_valor}}"42"Campo "valor" do payload
{{webhook_acao}}"cadastrar"Campo "acao" do payload
{{webhook_payload}}{...}Dict completo
💡 Regra: cada campo de primeiro nível vira webhook_CAMPO. Campos aninhados ficam no webhook_payload.

🌍 PARTE 4 — Expor para a internet (ngrok)

Para que sistemas externos (Zapier, Make, GitHub, etc.) consigam chamar o seu PyFlow, você precisa expor a porta 8080 para a internet.

Usando ngrok (gratuito, 5 min para configurar)

1
Baixe o ngrok em ngrok.com/download

Crie conta gratuita, extraia o .exe na pasta que quiser.

2
Abra PowerShell na pasta do ngrok e rode: 
.\ngrok.exe http 8080
3
Copie a URL pública gerada. Exemplo: 
https://abc123.ngrok-free.app
4
Agora qualquer sistema externo pode chamar: 
POST https://abc123.ngrok-free.app/webhook/meu_fluxo

E o PyFlow na sua máquina vai executar o fluxo 🚀

⚠️ A URL do ngrok muda a cada sessão (no plano gratuito). Para URL fixa, use o plano pago ou hospede o PyFlow em um servidor.
